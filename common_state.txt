**추가 작성 예정
이클립스 깃 설정법 다시 안내 합니다. 확인 부탁요.

- 깃을 새로 시작하는 경우나 작업 환경이 바뀔 경우.
 1. 이클립스 marketplace 에서  Egit패키지를 설치한다.

 2. 설치 후 오른쪽 상단 Open Perspective(Quick access 옆 버튼)을 누르고 Git을 선택

 3. Git Repository 탭에서 'Clone Repository'를 선택하고 git 주소 및 계정 입력

 4. Repository import 완료 후 하단 history 탭에서 Refresh(왼쪽 제일 끝)와 
 Show All Branches and Tags(오른쪽 끝 화살표 갈래모양)을 눌러서 현재 브랜치 현황을 본다.

 5. 자신이 기존에 다른 곳에서 작업하던 브랜치가 있으면 해당 브랜치에서 create branch를 눌러 동일한 이름으로 브랜치를 생성하거나, 
 merge한 기록이 있으면 해당 브랜치에서 새브랜치를 생성하여 계속 작업한다.
 
 6. 프로젝트 작업은 'widow->show view->others->project explorer'를 선택하여 'import -> git -> localrepository'를 선택하고 작업하면 된다.

- 간단한 깃 작업 방법(쉽습니다.)
 1. 깃 작업을 시작할때는 다른사람의 푸쉬 현황이나 머지현황을 보기 위해 '프로젝트 우클릭-> team -> fetch from upstream' 을 클릭하여(git repository 탭에서는 바로 fetch from upstream 실행)
 작업 현황일 보고 필요할 경우 브랜치를 생성하여 계속 작업한다.
 
 2. 깃 작업이 완료 된 시점에서는 커밋과 푸쉬를 하는 습관을 기르자(저장 안하면 모든것이 끝이야) 
 **참고 - 커맨드 라인(또는 맥 이나 리눅스 shell)에서는 특정 작업 후 add를 하고, add를 commit 이후 push이나, IDE에서는 변경 작업이 생기면 자동 add가 실행되고 커밋과 푸쉬를 한번에 진행할 수 있다.
 (생활코딩 깃 강의를 참조해보세용)
 
 3. 커밋 및 푸쉬때는 다른사람이 어떤 작업이 진행되었는지 알수 있도록 간단한 메시지를 남겨놓는다. 중요하다면 길게 남겨도 무방하다.
 
 자 궁금하면 개별적으로 연락을 주시거나, 또는 깃에 대해 좋은 팁을 찾으셨으면 여기에 이어서 작성해 주세요!!!
